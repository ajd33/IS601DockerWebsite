When it comes to computing, virtualization most commonly refers to virtual machines (VM). A virtual machine is a software representation of a physical computing machine like a smartphone or laptop computer. For example, if you have a Macintosh computer and need to run Windows only programs, you can accomplish this by using a virtual machine program like VMware to create a “virtual Windows computer” inside of your host Macintosh machine and then run the Windows only program inside of that VM. From the Windows operating system’s perspective, it’s running normally on dedicated hardware when in reality it’s really sharing hardware.

You can run multiple VM’s simultaneously on a host computer and partition resources between them. If you want to and your hardware allows, you can run 2 VM’s running Ubuntu and 1 VM of windows on one machine. VM’s also allow hardware resources to be partitioned accordingly. If your host machine has 8GB of RAM, you can assign 2GG to each Linux VM and 4GB to the Windows VM. Some VM host software have a hypervisor which is a can dynamically allocate hardware to each VM as needed.

One big advantage of VM’s is that you can save the state as a file. This means you can have multiple version of the same VM, let’s say a fresh install of Windows with no third party programs installed and another VM of the same Windows operating system, but with highly customized. Or let’s say you just configured a Linux VM running apache server and you need to have redundant copies of the server running on multiple machines. You can do this by saving the state on the first machine then deploying the saved state to other physical machines.

Another key aspect of VM’s is isolation. This has a few advantages. One advantage is that if the VM operating system crashes, it will not crash the host machine OS. The operating systems, even though they’re sharing the same hardware, can not read each other’s files. For security purposes this isolation is ideal, if a VM gets a virus then it can not share this virus with the host or other VM’s.
